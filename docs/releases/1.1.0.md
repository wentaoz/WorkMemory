# WorkMemory 1.1.0

WorkMemory 1.1.0 turns captured context into a practical daily workflow: review grouped work sessions, promote durable memories, ask questions with traceable evidence, and move decisions into action.

## What's new

- **Daily work sessions** group repeated window, browser, typing, and OCR captures into a clear timeline instead of flooding the memory list.
- **Ask Memory with citations** combines local semantic search and full-text matching, supports date and project scopes, and links every evidence item back to its source.
- **Projects and focus** organize memories and sessions by project, with pinned memories surfaced on the Today view.
- **Action workflow** adds priority, status, due dates, editing, deferral, and manual one-way export to a dedicated Apple Reminders list.
- **Document knowledge** indexes supported local documents in chunks with page, slide, or sheet locators for more precise retrieval.
- **Reliable summaries** add layered processing for larger days, progress, cancellation, run history, automatic catch-up, configurable schedule time, and API connection testing.
- **Safer first run** keeps passive capture off until the user opts in and exposes data, permissions, indexing, and model status in Data Health.

## Improvements and fixes

- Database schema v2 preserves existing passive-capture history as archived activities and creates a timestamped pre-migration backup.
- Startup now loads bounded recent records, reducing memory use on long-running workspaces.
- Passive capture waits for a stable foreground context, refreshes long-running sessions, and avoids indexing archived legacy activity.
- Updated records no longer leave duplicate full-text index entries.
- Interrupted summary jobs are recovered with an explicit failed state instead of appearing to remain active.
- Existing memory and action data remains compatible with the expanded models.
- Model requests have bounded timeouts and retry transient network and service failures.

## Compatibility

- macOS 13 or later.
- Universal build for Apple Silicon and Intel Macs.
- Existing WorkMemory databases are migrated automatically on first launch. A backup is stored under `WorkMemory/Backups` before migration.

## Installation

Drag `WorkMemory.app` to Applications. This public build is signed with an Apple Development identity but is **not notarized**. If macOS blocks the first launch, right-click the app and choose **Open**.

The SHA-256 checksum is published alongside the DMG asset.
